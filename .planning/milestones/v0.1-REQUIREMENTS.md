# Requirements: GameMotion

**Defined:** 2026-01-24
**Core Value:** JSON-to-video rendering engine must work reliably — everything else builds on this foundation

## v1 Requirements

Requirements for initial release. Each maps to roadmap phases.

### Rendering

- [x] **RNDR-01**: User can add text elements with font family, size, weight, style, color, alignment
- [x] **RNDR-02**: User can add text shadow and stroke/outline effects
- [x] **RNDR-03**: User can add text background with padding and border radius
- [x] **RNDR-04**: User can set text max width for automatic wrapping
- [x] **RNDR-05**: User can animate text word-by-word for caption-style reveals
- [x] **RNDR-06**: User can add image elements from URLs or uploaded assets
- [x] **RNDR-07**: User can set image fit mode (cover, contain, fill)
- [x] **RNDR-08**: User can add border radius to images
- [x] **RNDR-09**: User can add shape elements (rectangle, circle, ellipse, line)
- [x] **RNDR-10**: User can set shape fill color including linear/radial gradients
- [x] **RNDR-11**: User can set shape stroke color and width

### Animation

- [x] **ANIM-01**: User can animate element properties with keyframes (position, scale, rotation, opacity)
- [x] **ANIM-02**: User can specify easing function for animations (12 functions: linear, ease family, bounce, elastic)
- [x] **ANIM-03**: User can apply enter animation presets (fade, slide, scale, bounce)
- [x] **ANIM-04**: User can apply exit animation presets (fade, slide, scale, bounce)
- [x] **ANIM-05**: User can control animation timing with start time and duration

### Scenes

- [x] **SCEN-01**: User can create multiple scenes with different backgrounds
- [x] **SCEN-02**: User can set scene duration
- [x] **SCEN-03**: User can apply transitions between scenes (fade, slide, zoom)
- [x] **SCEN-04**: User can configure transition duration and easing

### Audio

- [x] **AUDI-01**: User can add background audio track to video
- [x] **AUDI-02**: User can control audio volume (0-1)
- [x] **AUDI-03**: User can apply audio fade in at start
- [x] **AUDI-04**: User can apply audio fade out at end

### API

- [x] **API-01**: Developer can authenticate with API key in Authorization header
- [x] **API-02**: Developer can submit render job via POST /render with JSON spec
- [x] **API-03**: Developer can poll job status via GET /render/:jobId
- [x] **API-04**: Developer can receive sync response for short videos (<30s)
- [x] **API-05**: Developer can configure webhook URL to receive job completion notification
- [x] **API-06**: Developer can retrieve validation errors for invalid JSON specs
- [x] **API-07**: Developer is rate-limited based on plan (Free: 10/min, Pro: 60/min)

### AI & Templates

- [x] **TMPL-01**: User can generate JSON template from natural language description
- [x] **TMPL-02**: User can specify target platform (TikTok, YouTube, Instagram) in AI generation
- [x] **TMPL-03**: User can specify video style (energetic, professional, playful) in AI generation
- [x] **TMPL-04**: AI returns list of {{variables}} requiring user input
- [x] **TMPL-05**: User can substitute {{variables}} with custom data when rendering
- [x] **TMPL-06**: User can list available built-in templates via API
- [x] **TMPL-07**: User can render using built-in template by ID

### Assets

- [x] **ASST-01**: Developer can upload image assets via POST /assets
- [x] **ASST-02**: Developer can upload audio assets via POST /assets
- [x] **ASST-03**: Developer can reference uploaded assets by ID in render spec
- [x] **ASST-04**: System caches remote assets during render session

### Output

- [x] **OUTP-01**: System outputs MP4 video with H.264 encoding
- [x] **OUTP-02**: User can configure output dimensions (up to 1920x1920)
- [x] **OUTP-03**: User can configure output fps (1-60, default 30)
- [x] **OUTP-04**: User can configure total video duration (up to 300s)
- [x] **OUTP-05**: Rendered video is accessible via URL for download
