  <behavior>
  The validateVideoSpec function accepts unknown input and returns a discriminated union:

  **Success case:**
  ```typescript
  validateVideoSpec({ output: { width: 1920, height: 1080, fps: 30, duration: 60 } })
  // => { success: true, data: { output: { width: 1920, height: 1080, fps: 30, duration: 60 } } }
  ```

  **Default fps applied:**
  ```typescript
  validateVideoSpec({ output: { width: 1920, height: 1080, duration: 60 } })
  // => { success: true, data: { output: { width: 1920, height: 1080, fps: 30, duration: 60 } } }
  ```

  **Validation failure - exceeds limits:**
  ```typescript
  validateVideoSpec({ output: { width: 2000, height: 1080, duration: 60 } })
  // => { success: false, error: { message: "...", fieldErrors: { "output.width": ["Width cannot exceed 1920 pixels"] } } }
  ```

  **Validation failure - missing required:**
  ```typescript
  validateVideoSpec({ output: { width: 1920 } })
  // => { success: false, error: { message: "...", fieldErrors: { "output.height": [...], "output.duration": [...] } } }
  ```

  **Test cases (comprehensive):**
  - Valid: minimum values (1x1, 1fps, 0.001s duration)
  - Valid: maximum values (1920x1920, 60fps, 300s)
  - Valid: default fps applied when omitted
  - Invalid: width > 1920
  - Invalid: height > 1920
  - Invalid: fps > 60
  - Invalid: fps < 1
  - Invalid: duration > 300
  - Invalid: duration <= 0
  - Invalid: non-integer width/height/fps
  - Invalid: missing required fields (width, height, duration)
  - Invalid: wrong types (string instead of number)
  - Invalid: null/undefined input
  - Multiple errors: return all field errors, not just first
  </behavior>
