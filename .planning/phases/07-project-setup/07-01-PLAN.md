<task type="auto">
  <name>Task 1: Scaffold Vite React TypeScript project</name>
  <files>
    studio/package.json

    studio/tsconfig.app.json

    studio/vite.config.ts

    studio/src/main.tsx
    studio/src/App.tsx
    studio/src/index.css
    studio/src/vite-env.d.ts
  </files>
  <action>
    Create the /studio directory and scaffold a Vite React TypeScript project.

    1. Run: `npm create vite@latest studio -- --template react-ts`

    2. Navigate to studio directory and install base dependencies:
       `cd studio && npm install`

    3. Install Tailwind v4 with Vite plugin:
       `npm install tailwindcss @tailwindcss/vite`

    4. Install shadcn dependencies:
       `npm install lucide-react class-variance-authority clsx tailwind-merge`

    5. Update studio/vite.config.ts to:
       - Import and add @tailwindcss/vite plugin
       - Add path alias for '@' pointing to './src'
       - Configure proxy for '/studio/*' routes to 'http://127.0.0.1:3000' (CRITICAL: use 127.0.0.1, NOT localhost - Node 17+ IPv6 issue)
       - Also proxy '/render', '/generate', '/templates', '/health' routes
       - Set build.outDir to '../dist/studio'

    6. Update studio/tsconfig.json to add baseUrl and paths for '@/*' alias

    7. Update studio/tsconfig.app.json to also include baseUrl and paths

    8. Replace studio/src/index.css content with just: `@import "tailwindcss";`
       (Tailwind v4 requires only this single import, no @tailwind directives)

    9. Create studio/src/lib/utils.ts with cn() helper:
       ```typescript
       import { type ClassValue, clsx } from "clsx"
       import { twMerge } from "tailwind-merge"

       export function cn(...inputs: ClassValue[]) {
         return twMerge(clsx(inputs))
       }
       ```

    10. Update studio/src/App.tsx to a simple placeholder with Tailwind classes:
        - Show "GameMotion Studio" heading with Tailwind styling
        - Add a simple card layout to verify Tailwind is working
        - Include text like "API Status: Checking..." to test proxy later

    11. Initialize shadcn: `npx shadcn@latest init`
        When prompted:
        - Style: Default
        - Base color: Slate
        - CSS variables: Yes
        - React Server Components: No
        - Write to components.json: Yes
        (shadcn creates components.json for future component additions)

    IMPORTANT: After vite.config.ts changes, server requires restart. Test after all config changes complete.
  </action>
  <verify>
    1. Run `cd studio && npm run dev` - should start on port 5173
    2. Open http://localhost:5173 in browser - should see App content
    3. Inspect element - Tailwind classes should apply (check computed styles)
    4. Check terminal - no errors during startup
  </verify>
  <done>
    - Vite dev server starts without errors on port 5173
    - React renders in browser with Tailwind-styled placeholder
    - Proxy configured (will return 502 until backend routes exist - expected)
    - shadcn initialized with components.json present
  </done>
</task>
