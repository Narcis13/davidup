<task type="auto">
  <name>Task 1: Add template CRUD routes to backend</name>
  <files>src/api/routes/studio.ts</files>
  <action>
Add POST, PATCH, and DELETE routes for studio templates to the existing studioRoutes:

1. POST /templates - Create template
   - Accept JSON body: { name: string, spec: object, conversationId?: string }
   - Generate UUID for id
   - Insert into studio_templates table with JSON.stringify(spec)
   - Return created template object

2. GET /templates/:id - Get single template (for detail view)
   - Fetch by id from studio_templates
   - Return 404 if not found

3. PATCH /templates/:id - Update template (rename)
   - Accept JSON body: { name: string }
   - Update name and updated_at in studio_templates
   - Return 404 if not found
   - Return updated template object

4. DELETE /templates/:id - Delete template
   - Delete from studio_templates by id
   - Return 204 No Content on success

Use db.prepare().run() for mutations, db.prepare().get() for single row.
Follow existing conversation routes pattern in the same file.
  </action>

<task type="auto">
  <name>Task 2: Create templates.ts API hooks</name>
  <files>studio/src/api/templates.ts</files>
  <action>
Create TanStack Query hooks for template CRUD operations following the pattern in chat.ts:

1. Define types:
   - StudioTemplate interface (id, name, spec, conversation_id, created_at, updated_at)
   - CreateTemplateInput (name, spec, conversationId?)
   - UpdateTemplateInput (id, name)

2. Define query keys:
   - templateKeys.all, templateKeys.list(), templateKeys.detail(id)

3. Create hooks:
   - useTemplates() - Query to fetch all templates with 5-minute staleTime
   - useTemplate(id) - Query to fetch single template (enabled when id truthy)
   - useCreateTemplate() - Mutation with cache invalidation on success
   - useUpdateTemplate() - Mutation with optimistic update (update name in cache immediately)
   - useDeleteTemplate() - Mutation with optimistic update (remove from cache immediately)

For optimistic updates:
- In onMutate: cancel queries, get previous data, update cache optimistically
- In onError: rollback to previous data
- In onSettled: invalidate queries to refetch

Follow the exact pattern from 09-RESEARCH.md Pattern 2.
