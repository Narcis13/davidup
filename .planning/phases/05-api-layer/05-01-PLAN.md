<task type="auto">
  <name>Task 2: Create Hono app skeleton with health check</name>
  <files>
    - src/api/app.ts
    - src/api/middleware/error-handler.ts
    - src/api/server.ts
  </files>
  <action>
Create `src/api/middleware/error-handler.ts`:
- Export errorHandler function compatible with Hono's app.onError signature
- Handle HTTPException from 'hono/http-exception' - return its response
- Handle ZodError - return 400 with fieldErrors parsed from error.issues
- Handle generic Error - log to console.error, return 500 with { error: 'Internal server error' }
- Return proper JSON content-type

Create `src/api/app.ts`:
- Import Hono, cors from 'hono/cors', logger from 'hono/logger'
- Create typed Hono app with Variables type: { userId: string; plan: 'free' | 'pro' }
- Apply global middleware: logger(), cors() with permissive config for MVP
- Apply errorHandler using app.onError()
- Add GET /health route returning { status: 'ok', timestamp: new Date().toISOString() }
- Export app

Create `src/api/server.ts`:
- Import serve from '@hono/node-server'
- Import app from './app.js'
- Read PORT from process.env (default 3000)
- Start server with serve({ fetch: app.fetch, port })
- Add SIGTERM handler for graceful shutdown: server.close(() => process.exit(0))
- Log "Server running on http://localhost:{port}" on startup
  </action>
  <verify>
- `npx tsc --noEmit` passes
- `node --import tsx src/api/server.ts &` starts server
- `curl http://localhost:3000/health` returns `{"status":"ok","timestamp":"..."}`
- Kill server with SIGTERM
  </verify>
  <done>
Hono app responds to GET /health. Error handler catches exceptions and returns JSON errors. Server starts with graceful shutdown.
  </done>
</task>
