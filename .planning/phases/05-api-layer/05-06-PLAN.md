  <action>
Create GET /download/:jobId endpoint to serve rendered video files:

1. Create `src/api/routes/download.ts` with Hono router:
   - Import Hono, createReadStream (from node:fs), stat (from node:fs/promises)
   - Export `downloadRoutes` as Hono instance

2. Implement GET /:jobId handler:
   - Extract jobId from path parameter (note: may include .mp4 extension)
   - Strip .mp4 extension if present: `jobId.replace(/\.mp4$/, '')`
   - Construct file path: `outputs/${jobId}.mp4`
   - Check file exists with stat(), return 404 JSON if not found
   - Create read stream and return with proper headers:
     - Content-Type: video/mp4
     - Content-Disposition: attachment; filename="{jobId}.mp4"

3. Pattern to follow (from VERIFICATION.md recommendation):
```typescript
import { Hono } from 'hono';
import { createReadStream } from 'node:fs';
import { stat } from 'node:fs/promises';

export const downloadRoutes = new Hono();

downloadRoutes.get('/:jobId', async (c) => {
  const rawJobId = c.req.param('jobId');
  const jobId = rawJobId.replace(/\.mp4$/, '');
  const filePath = `outputs/${jobId}.mp4`;

  try {
    await stat(filePath);
  } catch {
    return c.json({ error: 'Video not found' }, 404);
  }

  const stream = createReadStream(filePath);
  return c.body(stream, {
    headers: {
      'Content-Type': 'video/mp4',
      'Content-Disposition': `attachment; filename="${jobId}.mp4"`,
    },
  });
});
```

Note: Download endpoints are intentionally PUBLIC (no auth middleware) to allow direct browser downloads and sharing via URL. This matches common video hosting patterns.
