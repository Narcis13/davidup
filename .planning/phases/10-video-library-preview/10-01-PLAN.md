
<task type="auto">
  <name>Task 2: Add video routes to studio API</name>
  <files>src/api/routes/studio.ts, src/api/services/studio-db.ts</files>
  <action>
Install the `open` package first: `npm install open`

Add thumbnail_path column to videos table in studio-db.ts (after status column):
```sql
thumbnail_path TEXT,
```

Extend studio.ts with video routes. Import at top:
```typescript
import open from 'open';
import * as fs from 'node:fs/promises';
import * as path from 'node:path';
import { generateThumbnail, getVideoMetadata } from '../services/video-service.js';
```

Add these routes:

1. **GET /videos** - List videos with optional template filter
   - Query param: `templateId` (optional)
   - JOIN with studio_templates to get template_name
   - Order by created_at DESC
   - Return array of videos with template_name included

2. **POST /videos/:id/open** - Open video in system player
   - Get file_path from database
   - Resolve to absolute path with path.resolve()
   - Call `await open(absolutePath)`
   - Return { success: true } or 500 error

3. **DELETE /videos/:id** - Delete single video
   - Get file_path and thumbnail_path from database
   - Delete files (ignore errors - DB cleanup is primary)
   - Delete from database
   - Return 204 No Content

4. **POST /videos/delete-batch** - Batch delete videos
   - Accept JSON body: { ids: string[] }
   - Validate ids array is present and non-empty
   - Get file paths for all videos
   - Delete files (best effort)
   - Delete from database
   - Return { deleted: number }

5. **GET /thumbnails/:filename** - Serve thumbnail images
   - Read file from outputs/thumbnails/{filename}
   - Set Content-Type: image/jpeg
   - Set Cache-Control: public, max-age=86400
   - Return file buffer or 404

6. **POST /templates/:id/render** - Trigger render from template
   - Get template spec from database
   - Parse spec JSON
   - Create job ID with crypto.randomUUID()
   - Create video record with status='rendering'
   - Import and use existing JobQueueService (may need to create singleton instance)
   - Return { jobId: string }

7. **GET /render/:jobId** - Get render job status
   - Get job from JobStore (via JobQueueService)
   - Return job status, progress, error, video_id
  </action>
