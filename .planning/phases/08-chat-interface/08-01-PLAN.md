<task type="auto">
  <name>Task 1: Create streaming chat endpoint</name>
  <files>src/api/routes/chat.ts</files>
  <action>
Create a new Hono router at `src/api/routes/chat.ts` with a POST `/chat` endpoint that:

1. Accepts JSON body: `{ conversationId?: string, message: string, history?: Array<{role, content}> }`

2. Creates new conversation if `conversationId` not provided:
   - Generate UUID with `crypto.randomUUID()`
   - Insert into `conversations` table with title = first 50 chars of message
   - Return new conversationId in the SSE stream

3. Saves user message to `messages` table:
   - Generate UUID for message id
   - Insert with `conversation_id`, `role: 'user'`, `content`

4. Builds messages array for OpenRouter:
   - System prompt explaining the assistant is a video template generator for GameMotion
   - Include template format explanation with VideoSpec JSON structure
   - Include conversation history from `history` param
   - Add current user message

5. Calls `callOpenRouter` from `ai-client.ts`:
   - Model: `process.env.OPENROUTER_MODEL ?? 'anthropic/claude-sonnet-4'`
   - Request JSON response format
   - Max tokens: 4096

6. Returns SSE stream using Hono's `streamSSE`:
   - For MVP, simulate streaming by chunking the response (true streaming can be added later)
   - Send `{ type: 'chunk', content: '...' }` events
   - On completion, save assistant message to database
   - Send `{ type: 'done', conversationId, messageId, template }` event
   - On error, send `{ type: 'error', message: '...' }` event

System prompt should explain:
- Assistant generates valid VideoSpec JSON for GameMotion
- Use `{{variableName}}` syntax for user-customizable content
- For refinement requests, modify the previous template
- Response can include brief explanation before JSON

Import `streamSSE` from `hono/streaming` (not `hono/helpers/streaming`).
  </action>
  <verify>
TypeScript compiles without errors: `npx tsc --noEmit`
  </verify>
  <done>
chat.ts file exists with chatRoutes exported, POST /chat endpoint implemented with SSE streaming.
  </done>
</task>
